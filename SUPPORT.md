## Asking Questions

### GitHub Discussions

For questions not answered by documentation:

1. Go to the **Discussions** tab
2. Check if your question was already asked
3. Open a new discussion if not

**Good questions include:**
- What you're trying to accomplish
- What you've already tried
- Specific context about your situation

### What We Can Help With

- Clarifying content in this repository
- Guidance on which resources to use
- Suggestions for additional topics to cover
- General AI usage questions

### What We Cannot Help With

- Provider-specific API issues (contact the provider)
- Debugging your specific code
- Questions unrelated to this repository's scope

---

## Reporting Issues

### Content Issues

If you find:
- Errors or inaccuracies
- Outdated information
- Broken links
- Unclear explanations

Open an **Issue** with:
- Location of the problem (file and section)
- What's wrong
- Suggested correction (if you have one)

### Suggestions

For new content ideas or improvements:

1. Open an **Issue** with the "enhancement" label
2. Describe what you'd like to see
3. Explain why it would be valuable

---

## Contributing Fixes

The fastest way to fix issues is to contribute directly:

1. Fork the repository
2. Make your changes
3. Submit a pull request

See [CONTRIBUTING.md](CONTRIBUTING.md) for guidelines.

---

## Response Times

This is a community resource maintained by volunteers.

- **Issues:** Typically reviewed within a week
- **PRs:** Typically reviewed within 1-2 weeks
- **Discussions:** Community members may respond faster

Complex issues may take longer. Patience appreciated.

Be respectful, constructive, and helpful. We're all here to learn and improve.
