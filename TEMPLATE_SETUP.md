## Quick Start

This template is designed to help you create a stunning GitHub profile README with all the supporting documentation. Follow these steps to customize it:

### Step 1: Update GitHub Username References

Replace all instances of `[GITHUB_USERNAME]` with your actual GitHub username:

- **README.md** (lines 44-45): Update stats dashboard links
  - Change `username=[GITHUB_USERNAME]` to `username=YOUR_USERNAME`
  - Change `user=[GITHUB_USERNAME]` to `user=YOUR_USERNAME`
- **README.md** (line 70): Update profile link
  - Change `https://github.com/[GITHUB_USERNAME]` to `https://github.com/YOUR_USERNAME`
- **SUPPORT.md** (lines 14, 21, 28): Update issue and discussion links
  - Change `https://github.com/[GITHUB_USERNAME]/[REPO_NAME]` appropriately

### Step 2: Update Repository References

Replace all instances of `[REPO_NAME]` with your repository name:

- **SUPPORT.md** (lines 14, 21, 28): Update links
  - For a profile README repo (special repo), use your username as the repo name
  - For other repositories, use the actual repository name

### Step 3: Add Your Contact Information

Replace placeholder contact information:

- **CODE_OF_CONDUCT.md** (line 59):
  - Replace `[YOUR_EMAIL_HERE]` with your contact email for conduct violations
- **SECURITY.md** (line 43):
  - Replace `[YOUR_EMAIL_HERE]` with your security report email
  - Replace `[YOUR_PGP_KEY_HERE]` with your PGP key fingerprint (or remove if you don't have one)

### Step 4: Customize Content

Edit the content sections to match your profile:

- **README.md**: Update skill icons and philosophy
- **CONTRIBUTING.md**: Adjust contribution guidelines as needed
- **CODE_OF_CONDUCT.md**: Customize community standards
- **SECURITY.md**: Update supported versions and policy details
- **SUPPORT.md**: Customize response times and support options

### Step 5: Upload to GitHub

1. Use this as a template repository when creating a new repo
2. Or manually update your existing profile repository with these files
3. Make sure to replace all placeholder values before committing

## Template Markers

Look for these markers to find all customization points:

- `[GITHUB_USERNAME]` - Your GitHub username
- `[REPO_NAME]` - Your repository name
- `[YOUR_EMAIL_HERE]` - Contact email addresses
- `[YOUR_PGP_KEY_HERE]` - PGP fingerprint
- `<!-- TEMPLATE_NOTE: ... -->` - Additional customization notes
